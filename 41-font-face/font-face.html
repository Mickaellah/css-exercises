1. Font file formates:
    • WOFF2
    • EOT
    • OTF
    • SVG
  2. WOFF and WOFF2 are good candidates to use in all browsers.
  3. The difference between @font-face and the other @-blocks are:
    • We don't need to give a name with @font-face and don't need to use querrie.
    • Some @font-face need licence whereas other @-blocks don't need.
    • @font-face cannot be declared within a CSS selector.
  4. Licencing restrictions
    This means that fonts cannot be used without permission from whose people who created the fonts.
  5. To supprort all those chosen fonts, we need to make the use of 8 font-face blocks
  6. Our users have to download 160KB in the example above.
  7. If we chose to offer WOFF2 as well as WOFF, we need to have 16 files to specify:
    Eg:
    @font-face { font-family: Roboto; src: url('./Roboto.woff2') format(woff2), url('./Roboto.woff') format(woff); }
  8. For us to be sure that user's browser doesn't download the fonts they might already have, we place the local() functon before placing url() to check whether the device already has the font saved locally, and if not, to download it from the source specified in the url();.
  9. Sometimes we have to assign fallback fonts for our custom webfonts we add via @font-face. We have do bear in mind that we should always use the same type the font for its fallback or not really different from its other.
  10. We can render fallback font firs or it is just blank untill downloaded, or by using font-display: ;to help the browser decide on which strategy to use.
