Width and height.

The answers of the quetions in the index.md.

  • The diffault values of:
    1. Min-width is 0.
    2. Max-width is none.
    3. Min-height is 0.
    4. Max-height is none.
  • The max-width overrides width and min-width overrides max-width. The max-height overrides height and min-height overrides max-height.
  • Even if your width value is less than min-width value, it will be fine, it won't have any effect since it will be overridden by the min-width value.
  • If the element is a block-like or an inline-like element the value of the width or height change.
  • When your min-width is greater than your max-width, it is still working but in the wrong way, I mean in the way than your are not mean it to be.
  • When widths are applied to table cells, and the content inside these cells doesn't fit, so it overflows and the text in the cell flows over its.
  • If you set the height to a numeric value as percentagrs and when a content of an element with it is to big and doesn't fit in the specified height, it will overflow.
  • The interesting characteristics with the ruleset in the index.md are the units of the two values are not the same and also the max-width is greater than the min-width.
  • The implied width of a div is the max-width.
