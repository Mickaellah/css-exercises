Table style

Unique properties associated with tables and each default value.

  1. Table-layout: auto;
  2. Border-spacing: 2px;
  3. Border-collapse: separate;
  4. Caption-side: top;
  5. Empty-cells: show;

Some values of properties and some html elements that correspond to them.

  • table display property value in css is similar to <table> HTML element.
  • table-row-group display value in css is similar to <tbody> HTML element.
  • table-header-group is similar to <thead> HTML element.
  • table-footer-group is the same as <tfoot> HTML element.
  • table-row is the same as <tr> HTML element.
  • table-cell is the same as <td> HTML element.
  • table-column-group is the same as <colgroup> HTML element.
  • table-column is behave like <col> HTML element.
  • table-caption is beheve like <caption> HTML element.

Answers of the questions in the index.md.

  1. We need to set the border-collapse property with its default value before the border-spacing sets and it will work effectively.
  2. Possible downsides of using the various table values for display on a set of elements are:
    1. Inconsistent handling by browsers.
    2. Design maintensnce later on.
  3. According to my research the differece between table-layout: fixed; and table-layout: auto; is
    • Fixed is used to set a fixed table layout and also the table and column widths are set by the widths of table and col or by the width of the first row of cells.
    • Auto is used to set the automatic table layout on the browser. This property sets the column width by unbreakable content in the cells.
  5. Empty-cells requires other declaration before it can work effectively and that is border-collapse property with its default value which is separate.
